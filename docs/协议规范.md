# DHP 协议规范

DHP 的目标是把人类设计材料转成 Agent 可执行的中间语言。

## 协议包结构

```txt
handoff/
  handoff.json
  pages/*.md
  components/*.md
  flows/*.md
  tokens/design-tokens.json
  mapping/component-map.shadcn.json
  api/api-contract.json
  acceptance/checklist.md
```

## Markdown 不是最终协议

Markdown 只是容器。真正要求 Agent 执行的是这些 JSON 代码块：

- `dhp-layout`
- `dhp-component`
- `dhp-binding`
- `dhp-interaction`
- `dhp-state-matrix`
- `dhp-responsive`
- `dhp-a11y`

## 组件映射原则

Agent 不能自行发明组件。抽象组件必须映射到 shadcn/ui 组合：

```json
{
  "RiskTable": {
    "implementedBy": ["Card", "Table", "Badge", "Button"]
  }
}
```

## Token 原则

视觉值必须来自 token 或语义变量。禁止在业务组件中写：

```tsx
<div style={{ color: "#123456", padding: 17 }} />
```

## 状态矩阵原则

每个页面至少覆盖：

- loading
- empty
- error
- no-permission
- success
