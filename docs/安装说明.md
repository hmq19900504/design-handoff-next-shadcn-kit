# 安装说明

## 环境要求

- Node.js 20+
- npm 10+
- Python 3.10+，仅 FastAPI 需要
- Codex CLI，可选，用于安装本地 plugin/skill

## 1. 本地校验协议

```bash
npm run verify:offline
```

这一步不需要安装第三方依赖，只验证 DHP 协议链路是否能跑通。

## 2. 安装依赖

```bash
npm install
```

## 3. 构建并同步 DHP Bundle

```bash
npm run dhp:validate
npm run dhp:build
npm run dhp:sync
```

## 4. 运行 Next.js 前端

```bash
npm run web:dev
```

访问：`http://localhost:3000`

## 5. 运行 Hono API

```bash
npm run api:hono:dev
```

访问：`http://localhost:8787/health`

## 6. 运行 FastAPI，可选

```bash
cd apps/api-fastapi
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

访问：`http://localhost:8000/docs`

## 7. 默认安装：通用 Skill

如果你要 `Codex` 和 `Claude` 统一使用同一份能力，默认先装通用 skill：

```bash
npm run skill:publish-local
```

这条主路径会同时完成：

- 运行关键契约测试
- 打包独立 skill 到 `dist/skills/design-handoff-next-shadcn`
- `Codex skill` -> `~/.codex/skills/design-handoff-next-shadcn`
- `Claude Code skill` -> `~/.claude/skills/design-handoff-next-shadcn`
- `AGENTS skill 镜像` -> `~/.agents/skills/design-handoff-next-shadcn`

说明：

- 这是默认推荐路径，因为 `Codex` 和 `Claude` 都支持。
- 当前统一接入状态可通过 `npm run tool:status` 查询。
- 如果你只想拆开执行，也可以单独运行 `npm run skill:pack` 和 `npm run tool:install`。

## 8. 可选：安装 Codex Plugin

本仓库已经提供 repo marketplace：

```txt
.agents/plugins/marketplace.json
```

在仓库根目录启动 Codex：

```bash
codex
/plugins
```

选择 marketplace：**DHP 本地插件市场**，安装 **DHP Next Shadcn**。

注意：这一节只对 `Codex` 生效，`Claude` 不支持 `.codex-plugin/plugin.json` 和 marketplace。

安装后新开对话，输入：

```txt
请使用 design-handoff-next-shadcn。
先补齐产品边界、对象模型、页面职责和交互状态。
默认输出 ASCII、Mermaid 和 DHP 协议块，再实现 Next.js/shadcn 前端与 Hono/FastAPI 后端。
不要发明未映射组件，不要把失败态伪装成空态。
```

## 9. 将最新仓库版本统一接入 Codex / Claude

如果你后续会持续更新本仓库，推荐使用统一安装入口，而不是只复制 skill 目录：

```bash
npm run skill:publish-local
```

这条命令默认会处理整条本地发布链：

- `test:design-handoff-skill`
- `test:skill-package`
- `test:tooling`
- 生成 `dist/skills/design-handoff-next-shadcn`
- `Codex skill` -> `~/.codex/skills/design-handoff-next-shadcn`
- `Claude Code skill` -> `~/.claude/skills/design-handoff-next-shadcn`
- `AGENTS skill 镜像` -> `~/.agents/skills/design-handoff-next-shadcn`

这样后续你更新本仓库后，新开的 `Codex` / `Claude` 会话会自动读取最新版本；不需要每次重新复制 skill。

如果你还想给 `Codex` 额外挂上 plugin/marketplace，再运行：

```bash
npm run skill:publish-local:codex
```

说明：

- 如果目标路径已存在且不是软链接，脚本会先备份，再改成软链接。
- 当前版本标识可通过 `npm run version:show` 查询。
- 如果要升级版本号，直接运行 `npm run version:sync -- 0.2.1`，会自动同步到 `package.json`、`plugin.json` 和 `SKILL.md`。
- 当前统一接入状态可通过 `npm run tool:status` 查询。
- 当前仓库内的 `CLAUDE.md -> AGENTS.md` 本身已经是仓库真源，不需要额外同步。
- 如果你显式启用了 `tool:install:codex`，脚本会先备份 `~/.codex/config.toml` 再写入 marketplace 配置。
