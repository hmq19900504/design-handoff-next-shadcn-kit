# 开源调研结论

截至本项目创建时，生态里有很多可拼装的能力，但没有一个开源项目完整覆盖以下闭环：

```txt
中文设计交接协议
+ Codex skill/plugin
+ Next.js/shadcn/ui 生产骨架
+ Hono/FastAPI 后端契约
+ token/component-map 校验
+ 可构建 bundle
```

## 可借鉴的开源/生态项目

| 项目/生态 | 价值 | 不足 |
|---|---|---|
| Storybook MCP | 让 Agent 读取真实组件、stories 和测试反馈 | 主要解决组件上下文，不定义设计交接协议 |
| Figma MCP | 给 Agent 读取 Figma 结构化上下文 | 仍需要项目内组件映射和代码规范 |
| shadcn/ui | Next/React/Tailwind 组件源码式复用 | 不解决设计协议、API 契约、状态矩阵 |
| shadcn/studio / blocks 生态 | 提供可复用 blocks 和 Figma-to-code 工作流 | 偏 block/模板，不等于企业内部 handoff 协议 |
| UI Builder | 使用真实 React 组件做可视化编辑并保存 JSON | 更像运行时 UI builder，不是 Codex skill + handoff 协议 |
| Hono RPC | 前后端 TypeScript 类型共享 | 只解决 API 类型，不解决 UI 设计语义 |
| FastAPI | 自动 OpenAPI 文档与 Python API | 后端能力强，但不解决前端设计还原 |

## 本项目的定位

本项目不是替代上述生态，而是把它们需要的“中间语言”补上：

- 用 DHP 定义设计语义。
- 用 tokens 定义视觉真值。
- 用 component-map 限制 Agent 组件选择。
- 用 api-contract 限制数据字段。
- 用 skill/plugin 固化 Codex 的执行流程。
